Introduction

GDScript is a high-level, dynamically typed programming language used to create various contents such as games, applications, animations and many more. Its syntax is somewhat similar to Python in terms of its blocks of code being indent-based, and many keywords are similar.

Its goal is to be optimized furthermore and integrated tightly with the Godot Engine. This will allow to expand its application in other fields aside from game development.


Coding

Despite being dynamically typed and being Python-like, its syntax resembles C or C++ more. Additionally, GDScript allows statically typed variables by providing its built-in types.

# Welcome to GDScript!
      var GDScript: String = "awesome"
      func _ready() -> void:
      	print("GDScript is " + GDScript + "!")
  • The sharp symbol # means that the whole line after it is a comment.
  • The variable was specified with its built-in type i.e. String class in this case.
  • The _ready() is built-in to the engine. It executes its code after the Node enters the scene.
    Since GDScript variable is a string, it will be added to the other string and will give this output:
GDScript is awesome!

Hello World

With the example earlier, we can see that writing Hello World! is just a walk at a park.

func _ready() -> void:
      	print("Hello World!")
Hello World!
  • The void tells the function that there are no values to be returned.

Variables

The variables can be class members (of the Node itself, therefore can be accessed by functions) or local to (inside of) functions. They are created with the var keyword. They may, optionally, be assigned a value upon initialization.

var a # Default value is 'null'
      var b = 25
      var c = 8.9
      var d = b + c # Variables are always initialized by order

Variables can optionally have a type specification. When a type is specified, the variable will be forced to have always that same type, and trying to assign an incompatible value will raise an error.

Types are specified in the variable declaration using a colon symbol : after the variable name, followed by the type.

var map_coordinates: Vector2
      var avatar_skin: Node = Sprite.new()

There is also a shorthand := for inferring the name type after initializing the variable:

var map_coordinates := Vector2()
      var avatar_skin := Sprite.new()

Type inference is only possible if the assigned value has a defined type, otherwise it will raise an error.

Valid types are:

  • Built-in types (int, String, Array, Vector2, etc.).
  • Engine classes (Node, Node2D, Control, Spatial, etc.).
  • Constant names if they contain a script resource (ThisScript if you declared const ThisScript = preload("res://this_script.gd")).
  • Other classes in the same script, respecting scope (MainClass.SubClass if you declared class SubClass inside the MainClass in the same scope).
  • Script classes declared with the class_name keyword.

match Statement
var value := 0.5
      for i in 5:
      	value += 0.25
      
      match(value):
      	1.5: 
      		print("This is not the value.")
      	1.75:
      		print("This is the right value.")
      	_:
      		print("This gets printed if none of the above were matched.")
"This is the right value."
